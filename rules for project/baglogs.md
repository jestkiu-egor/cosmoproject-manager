## 2026-04-14: onOpenAssistant is not defined в Sidebar
Симптом: При нажатии на "Ассистент" в боковом меню ошибка "Uncaught ReferenceError: onOpenAssistant is not defined"
Причина: Проп `onOpenAssistant` был объявлен в interface SidebarProps, но не был деструктурирован в функции компонента. В строке `export const Sidebar = ({ ... }: SidebarProps)` отсутствовало поле `onOpenAssistant`.
Решение: Добавил `onOpenAssistant` в деструктуризацию: `export const Sidebar = ({ activeTab, setActiveTab, projects, selectedProjectId, onSelectProject, onOpenAssistant }: SidebarProps)`
Предотвращение: Все пропсы из interface должны быть указаны в деструктуризации компонента. Проверять соответствие props между interface и фактическим использованием.

## 2026-04-14: Потеря файлов AI ассистента при merge в main
Симптом: После merge ветки jestkiu_vetka в main пропали файлы assistant_settings.sql, AssistantPage.tsx, KanbanBoard.tsx, llm.ts, types.ts. Sidebar не содержал раздел "Интеграции".
Причина: При использовании `git checkout --ours .` для разрешения конфликтов были взяты только файлы из remote main, а файлы из ветки (80f7674) которые должны были добавиться - были пропущены.
Решение: Восстановил файлы командой `git checkout 80f76744f3f27b88d41a04dbc5a1cee73082452b -- sql/assistant_settings.sql src/components/AssistantPage.tsx src/components/KanbanBoard.tsx src/lib/llm.ts src/types.ts`. Добавил недостающие изменения в Sidebar (раздел Интеграции) и App.tsx (подключение AssistantPage).
Предотвращение: При merge внимательно проверять все файлы. Использовать `git log --name-only` для проверки какие файлы должны были измениться. Не использовать `checkout --ours` без проверки.

## 2026-04-14: 401 Invalid API Key при работающем ключе
Симптом: Edge Function возвращала ошибку "API error: 401 - Invalid API Key", хотя ключ gsk_ был валидным и передавался в настройках.
Причина: Edge Function в Supabase использовала старый код без прокси и без передачи API ключа из настроек. Код в файле `supabase/functions/llm-proxy/index.ts` локально был обновлён, но не был задеплоен в Supabase.
Решение: Обновить Edge Function в Supabase Editor с актуальным кодом, который берет llm_api_key из settings и использует прокси.
Предотвращение: Всегда проверять что Edge Function задеплоены после обновления кода. Добавлять логирование в Edge Function для диагностики.

## 2026-04-14: AI кнопка создавала задачу сразу без проверки
Симптом: При нажатии кнопки AI задача создавалась сразу без возможности редактирования пользователем.
Причина: Код сразу создавал задачу при успешном ответе от LLM без открытия модального окна редактирования.
Решение: Изменить логику - при получении ответа от LLM заполнять поля формы (newTaskTitle, newTaskDescription, newTaskAssignee, newTaskDueDate, newTaskAmount) и открывать модалку create task через `setIsAddTaskModalOpen(true)` для ручной проверки.
Предотвращение: Всегда давать пользователю возможность проверить и отредактировать данные перед созданием сущности.

## 2026-04-14: Задача создавалась даже при неверном прокси
Симптом: При неверных настройках прокси задача все равно создавалась, просто с сырым текстом из буфера.
Причина: В handleAICreateTask при `parsed === null` создавалась задача с сырым текстом вместо показа ошибки.
Решение: Добавить проверки: если настройки не загружены - alert('Ошибка загрузки настроек'), если прокси не настроен - alert('Прокси не настроен'), если LLM вернул null - alert('Ошибка связи с нейронкой'). Задача создается только при успешном парсинге.
Предотвращение: Не создавать сущности при ошибках. Всегда показывать пользователю что произошла ошибка.
